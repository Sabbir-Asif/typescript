### üß† What is Namespace Augmentation?

**Namespace augmentation** means **adding more stuff to an existing namespace** ‚Äî like extending it from another file or later in the same file.

> Think of it like saying:
> ‚ÄúHey, that namespace already exists ‚Äî now I‚Äôm going to add more to it!‚Äù

---

### üì¶ Why Would You Use It?

* To **organize code** across multiple files
* To **add types/functions** to existing libraries or global objects
* To **extend third-party definitions** without modifying their original code

---

### üß± Basic Example

Let‚Äôs create a namespace and then **augment** it:

```ts
// logger.ts
namespace Logger {
  export function log(message: string) {
    console.log("LOG:", message);
  }
}
```

Now in another file (or later in the same file):

```ts
// logger-augment.ts

  export function warn(message: string) {
    console.warn("WARNING:", message);
  }
}
```

‚úÖ Now both functions exist:

```ts

Logger.warn("Careful"); // WARNING: Careful
```

You just **augmented** the `Logger` namespace with `warn`.

### üåç Real-World Use Case: Augmenting Global Libraries

Let‚Äôs say you‚Äôre using a third-party library that defines this:

```ts

```

You can augment it in your own `.d.ts` file:

```ts

  function farewell(): void;

```

Now both `greet()` and `farewell()` are part of `MyLibrary`.

### ‚ö†Ô∏è Key Rules

1. You must **use the same namespace name**.
2. You must use the `declare` keyword if it‚Äôs in a `.d.ts` file.
3. If you‚Äôre augmenting a module or global namespace, make sure TypeScript knows where to find it (via `tsconfig.json` or triple-slash references).
